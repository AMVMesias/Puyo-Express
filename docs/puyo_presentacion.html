Descripción de la App

Puyo Express es una plataforma integral de gestión de entregas y logística basada en una arquitectura cliente-servidor moderna.

Su función central es conectar restaurantes, clientes y repartidores, permitiendo creación de órdenes y seguimiento.

Objetivo DevSecOps

Garantizar que la información sensible esté protegida desde la persistencia (cifrado) hasta el acceso mediante autenticación robusta.

Asegurar la trazabilidad y reducir riesgos de filtración de información (Data Breach).

Límites de Confianza

Aislamiento Estricto

Subred Frontal (app_net): Expone únicamente el proxy inverso mapeado al puerto host público.

Subred de Datos (data_net): Totalmente aislada. El backend se comunica con la DB por el puerto 5432 internamente. Sin acceso público.

Protección de Contraseñas

Uso robusto de BCrypt con factor de trabajo 12 para evitar fuerza bruta y el uso de tablas arcoíris (salting automático).

@Bean
public PasswordEncoder passwordEncoder() {
    return new BCryptPasswordEncoder(12);
}

Mitigación de Fuerza Bruta

AuthRateLimitFilter: Máximo 10 intentos por cada 15 minutos en rutas de Auth. Bloqueo automático post-incidencia.

HTTP/1.1 429 Too Many Requests
{
  "error": "Too Many Requests"
}

Estructura de Logs

Registro estructurado de eventos relevantes sin comprometer la privacidad.

Se aplica sanitación rigurosa para evitar fuga de contraseñas, payloads completos y datos PII en consola.

Integración SIEM (JSON)

Uso de logstash-logback-encoder. Los logs se imprimen como un objeto JSON válido, ideal para Elasticsearch o Splunk.

{"@timestamp":"2026-07-20T15:35:12.456Z", "message":"Authentication failed for user: admin", "logger_name":"AuthRateLimitFilter", "level":"WARN", "client_ip":"192.168.1.45", "event_type":"AUTH_FAILURE"}

Hardening a Nivel Código

Cifrado PII

AES-256-GCM para columnas sensibles (Dirección, Teléfono) protegiendo el dato en reposo.

Uso de DTOs

Sanitación estricta y validación de entrada para evitar Inyecciones y asignación masiva de campos no deseados.

Manejo de Excepciones

La clase ApiExceptionHandler previene fugas de información interna (Information Disclosure).

Intercepta fallos y anula stacktraces crudos. El cliente siempre recibe un error HTTP estándar y limpio.

Anti-Fingerprinting

Ocultamiento de información del servidor para mitigar escaneos automatizados:

# nginx.conf
server_tokens off;
proxy_hide_header Server;

# application.yml (Spring)
server.server-header: ""
